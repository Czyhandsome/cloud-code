Commit message: fix(sandbox): add missing methods to sandbox-runtime stub to fix annotateStderr crash [chg-20260424-1034-qpo]

Summary: `SandboxManager.annotateStderrWithSandboxFailures` was undefined at runtime, crashing the Bash tool on every command execution. Root cause: `stubs/@anthropic-ai/sandbox-runtime/index.js` only stubbed 10 of the ~20 methods on `SandboxManager`. The adapter in `sandbox-adapter.ts` forwards many calls directly to `BaseSandboxManager` (the stub), so any missing method becomes an immediate crash at call site. Added all missing stub methods: `getAllowUnixSockets`, `getAllowLocalBinding`, `getEnableWeakerNestedSandbox`, `getProxyPort`, `getSocksProxyPort`, `getLinuxHttpSocketPath`, `getLinuxSocksSocketPath`, `waitForNetworkInitialization`, `getSandboxViolationStore`, `annotateStderrWithSandboxFailures`, `cleanupAfterCommand`.

- claude-code-source/node_modules/@anthropic-ai/sandbox-runtime/index.js (synced)

Notes: pnpm hard-links the stub into the virtual store, so updating `node_modules/@anthropic-ai/sandbox-runtime/index.js` also updated the pnpm store copy automatically. The `stubs/` directory is the canonical source; rebuilding with `bun install` would resync. Also note: `BashTool.tsx:710` passes `result.stdout` (not `result.stderr`) to `annotateStderrWithSandboxFailures` — likely a bug in the calling code, but harmless with the stub that just returns the input unchanged.
