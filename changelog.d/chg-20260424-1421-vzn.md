Commit message: fix(narrative): use dynamic fence length to prevent code-fence collision [chg-20260424-1421-vzn]

Summary: Narrative logs that embedded system prompts or tool results containing triple-backtick code fences would break the outer Markdown fence, corrupting rendering in Typora and other standard parsers. Added `fenceFor()` helper that counts the longest backtick run in content and uses one more backtick for the outer fence — the CommonMark-canonical approach. `formatTextBlock()` now uses this instead of a hardcoded ` ```text `.

Files:
- claude-code-source/src/utils/narrativeLogger.ts

Notes: This is the standard fix per CommonMark §4.5 — a fence is closed by the same character at equal or greater count, so always using N+1 backticks relative to content is spec-correct and renders well in Typora, GitHub, VS Code preview, and Pandoc.
